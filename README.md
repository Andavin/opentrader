## What it is

opentrader is a desktop app (built on [Tauri](https://tauri.app/)) that gives me one workspace to:

- View portfolios across **Alpaca**, **Robinhood**, and **Fidelity** with a single account-dropdown switch (Legend-style)
- Stream live quotes, charts, and options chains from whichever broker has the best data feed for the symbol
- Place orders (single-leg equities, multi-leg options, OCO/bracket) — paper or live
- Persist my widget layout across restarts
- Stay on top of P&L with an "auras" theme that subtly recolors the workspace based on session phase or day P&L

The brokers without official APIs (Robinhood, Fidelity) are driven through Playwright running on my own machine, so every request to those services originates from a real browser context with my real residential IP and cookies — the same fingerprint a human user produces. No datacenter proxying.

## Quick start

### Prereqs

| Tool          | Version | Install                                                                                                                                                       |
| ------------- | ------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Node.js       | ≥ 22    | [nodejs.org](https://nodejs.org/) or `nvm install 22`                                                                                                         |
| pnpm          | ≥ 10    | `corepack enable && corepack prepare pnpm@latest --activate`                                                                                                  |
| Rust + cargo  | stable  | [rustup.rs](https://rustup.rs/)                                                                                                                               |
| Tauri OS deps | varies  | macOS: `xcode-select --install` · Windows: WebView2 + VS C++ Build Tools · Linux: see [tauri.app/start/prerequisites](https://tauri.app/start/prerequisites/) |

### Run it

```bash
git clone <wherever-this-lives>
cd opentrader
pnpm install            # installs deps + builds the better-sqlite3 native binding
pnpm dev                # sidecar + tauri shell, in one terminal
```

The first Tauri compile pulls and builds ~150 Rust crates and takes 2–5 minutes cold. Subsequent runs are <10s.

If you want browser-only (no native shell, faster iteration cycle):

```bash
pnpm dev:web            # sidecar + Vite at http://localhost:1420
```

### Connecting Alpaca (the only broker fully wired today)

1. Sign up for a free Alpaca account at [alpaca.markets](https://alpaca.markets/)
2. In the dashboard: **Paper Trading → API Keys → Generate New Keys** (or **Live Trading → API Keys** if you want real money)
3. In opentrader, click the account dropdown (top-right) → **Connect Alpaca**
4. Paste the key ID + secret, leave **Paper trading** checked, **Connect**
5. The dropdown should show "● connected" plus a data-feed picker. Your tier auto-detects:
   - **Real-time SIP** if you have Algo Trader Plus
   - **Delayed SIP (15 min)** for free accounts (strictly better than IEX-only — full coverage with a delay)
   - **IEX only** as the universal fallback
6. Type a symbol in the Watchlist → click → the chart loads

Robinhood and Fidelity don't have working data paths yet — see the status table.

## Architecture (one diagram)

```
┌─ Tauri 2 desktop shell (Rust) ──────────────────────────────────┐
│  React 19 UI  ──►  KLineCharts + dockview + Tailwind v4         │
│      │                                                            │
│      │ HTTP+JSON (bearer)                                         │
│      ▼                                                            │
│  Node sidecar @ 127.0.0.1:1421 (Hono)                             │
│      ├─ broker-alpaca   (own REST + zod, no SDK dep)              │
│      ├─ broker-robinhood (Playwright persistent context)          │
│      ├─ broker-fidelity  (Playwright persistent context)          │
│      └─ SQLite (better-sqlite3 + Drizzle)                         │
└───────────────────────────────────────────────────────────────────┘
```

The sidecar binds only to **127.0.0.1** and requires a bearer token on every route except `/health`. It's intended for the same-machine frontend, not external access.

The brokers without official APIs (Robinhood, Fidelity) only ever talk to the network through Playwright's embedded Chromium, with persistent profiles stored at `~/.opentrader-data/profiles/<broker>`. Cookies survive restarts so the device-approval / login flow only runs once per broker.

## Project structure

```
apps/
  desktop/                  # Tauri app (Rust shell + React UI)
packages/
  broker-core/              # Broker interface + shared types + PaperBroker wrapper
  broker-alpaca/            # Alpaca adapter (own REST client + feed detection)
  broker-robinhood/         # Playwright scaffolding
  broker-fidelity/          # Playwright scaffolding
  db/                       # Drizzle SQLite schema
  sidecar/                  # Hono server, /broker + /layouts routes
```

For the full design + extension points, read **[CLAUDE.md](./CLAUDE.md)** — that file briefs future AI sessions on the architecture in more detail than I'd put in a public README.
